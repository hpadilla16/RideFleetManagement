Action Board — Mockup notes

Click "Mockup Notes" again para esconder.

Layout

  • 1920×1080, fullscreen kiosk en TV pared
  • 2 columnas: Pickups (60%) + Returns (40%)
  • Pickups tienen sub-section "Mañana AM" (06:00–12:00 día siguiente)
  • Header 88px · Footer 36px

Tema sobrio

  • Fondo casi-negro (#0a0a0d), no gradients
  • Cards con borde sutil, no sombras dramáticas
  • Acentos de color SOLO para estados accionables
  • Late returns: pulse rojo brillante (único accent fuerte) + border animado

Tipografía

  • Inter para UI general
  • JetBrains Mono para horas (alineación tabular)
  • Tamaños grandes (19–36pt) para legibilidad a 3m

Estados Pickup

  • Confirmed — futuro, neutral
  • Ready — cliente / contrato listo, mint
  • In Progress — pre-checkin abierto, azul
  • Checked Out — ya salió, verde tachado
  • No-Show — >30min late sin contacto, rojo + pulse
  • Cancelled — gris muy oscuro, tachado

Estados Return

  • Scheduled — futuro
  • Due Now — ±30min, amarillo
  • Late — pasó la hora, rojo + pulse
  • Returned — completado, verde tachado

Backend (próxima fase)

  • GET /api/public/store-board/:locationId/today
  • Auth: kiosk token (sin TTL, scoped a tenant + locationId)
  • Polling 30s · TTL 10s
  • Response: { pickups: [...], returns: [...], tomorrowAM: [...] }

Frontend (próxima fase)

  • Ruta: /store-board/:locationId?token=...
  • Sin AppShell · sin AuthGate
  • JS: reloj + countdown + auto-refresh

Cosas a confirmar contigo

  • "Counter A / Counter B" — ¿lo usan? Si no, lo quito
  • "Mañana AM" cutoff: 12:00 PM. ¿Te sirve o lo cortamos antes?
  • "No-Show" auto-trigger a los 30min: ¿correcto o es 60min?
  • ¿Quieres mostrar "Days remaining" en returns activos?
  • ¿Logo del tenant en el header (cuando el board sea por tenant)?
