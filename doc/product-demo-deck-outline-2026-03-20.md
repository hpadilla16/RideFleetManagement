# Product Demo Deck Outline 2026-03-20

## Goal

Create a presentation and live demo structure for prospects who want to understand what Ride Fleet can do today and what is coming next.

The presentation should sell one idea clearly:

- one platform
- one operations spine
- multiple business models

## Audiences

### 1. Rental Operator

Wants to see:

- reservations
- agreements
- payments
- reports
- commissions
- employee operations

### 2. Car Sharing Operator

Wants to see:

- hosts
- listings
- availability
- trips
- guest and host flows
- platform earnings

### 3. Dealership / Loaner Program Prospect

Wants to see:

- courtesy fleet operations
- service-lane style handoff
- insurance and liability handling
- inspections
- cost recovery
- employee mobility

## Demo Message

Ride Fleet is not just a rental tool.

It is a unified mobility operations platform that can support:

- classic rental
- car sharing
- dealership loaner / courtesy programs

without splitting the company into disconnected systems.

## Live Demo Flow

### Demo 1. Rental

1. Create booking from `/book`
2. Open confirmation
3. Complete pre-check-in
4. Review in back office
5. Show agreement / payment / checkout path

### Demo 2. Car Sharing

1. Open `/car-sharing`
2. Show host and listing
3. Create trip
4. Open linked workflow
5. Show `/guest` and `/host`

### Demo 3. Reporting And Controls

1. Open `/reports`
2. Show services sold
3. Show commission views
4. Show people & access

### Demo 4. Dealership Story
